# 🚀 Streaming ML Pipeline (MLOps MVP)

Проект реализует MVP системы машинного обучения для обработки **потоковых табличных данных** с поддержкой дообучения, мониторинга и инференса.

---

## 📌 Описание

Система имитирует работу production ML pipeline:

* данные поступают **батчами**
* модель обучается и **дообучается со временем**
* метрики качества отслеживаются **в динамике**
* доступен интерфейс для:

  * обучения (`update`)
  * предсказаний (`inference`)
  * отчётов (`summary`)

---

## 🏗️ Архитектура

Pipeline состоит из этапов:

1. **Data Ingestion** — разбиение данных на батчи
2. **Preprocessing** — обработка пропусков, кодирование, нормализация
3. **Feature Engineering** — генерация новых признаков (даты → длительность, возраст авто)
4. **Inference** — оценка модели на новых данных
5. **Training** — дообучение модели
6. **Monitoring** — логирование метрик
7. **Model Registry** — сохранение модели

## ⚙️ Установка (через uv)

```bash
uv venv
```

### 2. Установить зависимости

```bash
uv pip install -e .
```

---

## ▶️ Использование

### 🔹 1. Обучение (pipeline update)

```bash
uv run run.py -mode update -file data.csv
```

Что происходит:

* данные разбиваются на батчи
* модель обучается на первом батче
* далее:

  * оценивается на новом батче
  * дообучается
* метрики сохраняются

---

### 🔹 2. Предсказания (inference)

```bash
uv run run.py -mode inference -file new_data.csv
```

Результат:

```
predictions.csv
```

---

### 🔹 3. Отчёт (summary)

```bash
uv run run.py -mode summary
```

Вывод:

* статистика метрик (RMSE, MAE)

---

## 📊 Данные

Требования к датасету:

* ≥ 10 000 строк
* ≥ 10 признаков
* наличие временной переменной
* наличие пропусков

Пример признаков:

* характеристики автомобиля
* страховые параметры
* даты страхования

Целевая переменная:

```
claim_paid
```

---

## 🔄 Логика потоковой обработки

На каждом батче:

1. Модель делает предсказание
2. Считаются метрики
3. Модель дообучается
4. Метрики сохраняются

---

## 📈 Метрики

Используются:

* RMSE
* MAE

История сохраняется в:

```
metrics/metrics.csv
```

## 🧠 Особенности реализации

* preprocessing встроен в `sklearn Pipeline`
* поддержка категориальных и числовых признаков
* обработка дат:

  * длительность страховки
  * возраст автомобиля
* логирование метрик по батчам
